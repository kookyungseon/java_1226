0. 복습

static 
자바의 저장공간이 프로그램이 시작되고 끝까지 유지된다면 -> 정적이다 
실행도중에 생겨나거나 사라질 수 있다면 -> 동적이다 

static이 붙은 멤버들만 클래스 정보를 시점에 미리 메모리에 올라간다
static이 붙은 저장공간은 method영역이다 

static 키워드
- 멤버 변수 
	static이 붙은 변수 : static 변수, 클래스 변수, 정적 변수, …
	static이 안붙은 변수 : 인스턴스 

- 멤버 메소드의 종류 
	static이 붙은 메소드 : 정적 메소드,…
	안붙은 메소드 : 인스턴스 메소드 

1) static이 붙은 멤버들은 객체를 생성하지 않아도 사용할 수 있다
2) 객체와 상관없이 모든 객체가 공유해서 사용한다 
3) static 메소드는 인스턴스 멤버를 사용할 수 없다 
4) 클래스명.멤버명으로 사용한다!!


상속 
1. 기존에 선언된 클래스의 멤버(생성자는 제외)를 새롭게 만들 클래스에서 사용하고 싶을 때
  상속을 받고 추가할 기능만 작성한다

2. 여러 클래스 선언시 멤버 겹치는 경우, 부모 클래스 먼저 선언하고 자식 클래스에게 상속

상속문법
class Parents{
	Parents 멤버
}

class Child extends Parents {
	Parents, Child 멤버
}

Parents : 부모 클래스, 슈퍼 클래스, 기반 클래스, 상위 클래스
Child : 자식 클래스, 서브 클래스, 파생 클래스 , 하위 클래스 
*주의 사항
 자바에 상속은 단일상속만 지원한다 (자식 클래스는 1개의 부모 클래스를 가진다)

오버라이딩
부모 클래스에서 선언한 메소드를 자식 클래스에서 수정하고자 할때 
부모 클래스와 동일한 이름으로 선언, 재정의 

super
super : 부모의 참조값
	 부모의 멤버에 접근할 때 사용한다 .
	
super( ) : 부모의 생성자
	    자식 클래스 타입의 객체로 부모 필드에 접근할 수 있다. 
	    자식 클래스는 부모 클래스 없이는 존재할 수 없다.
	    그러므로  자식의 생성자는 항상 부모의 생성자를 호출한다.  
	    즉, 자식 객체를 생성하게 되면 부모 객체도 생성된다 ( 부모 객체가 먼저 생성됨)
	    부모 생성자를 호출하는 방법은 supter( ) 를 사용하면 된다 
	    만약, super()를 작성하지 않더라도 컴파일러가 자동으로 작성해준다 .
