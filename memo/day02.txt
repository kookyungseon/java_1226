Java의 기본 구조
1) JVM (Java Virtual Machine)
: 운영체제에서 맞춰서 java로 만든 프로그램을 실행시켜 준다

2) JRE (Java Runtime Environment)
: JVM을 실행한다

3) JDK(Java Development kit)
: JRE
개발에 필요한 도구들을 가지고 있다 

4) IDE(통합 개발 환경)
: 개발의 편의성을 높이기 위해서 사용하는 도구 
Ex) 이클립스 : 무료이다, 다양한 단축키 제공하여 개발의 편의성을 높여준다 

1) 이클립스 글씨체 변경

window > Preferences > font 검색 > Colors and Fonts 클릭 > Basic
> Text Font > Edit 설정 Consolas > Apply

2) 인코딩 타입 변경

window > Preferences > enc 검색 > Workspace 하단의 Text file encoding UTF-8 > Apply

3) 컴파일러 변경

window > Preferences > compiler 검색 >  JDK Compliance에서 level 변경
(기본 18 -> 17로 변경) > Apply

5) 이클립스 우측 상단의 창문 모양 클릭 > JAVA 선택 > Open

4. 자바의 구성요소 
프로젝트 > 패키지 > 클래스 > 매소드 > 소스코드(명령어) 

프로젝트 폴더 생성 
단축키 : cmd + n > java project  선택 
프로젝트 이름 설정, 모듈 체크 해제 > finish(엔터)

패키지 생성

Src > 마우스 좌클릭 > cmd +n > package 검색후 >  next

클래스 
Package > 마우스 좌클릭 > cmd + n > class 검색 후 > 클래스 이름 (앞글자를 대문자로) >finish 

메소드 생성
1. 클래스의 중괄호 {} 안에서 main이라고 쓰고 opt + space < 엔터(자동완성 기능)
2. 클래스를 생성할 때 main 메소드 체크 > finish

변수 
어떤 값을 저장하기 위한 저장 공간 

1) 변수를 만드는 방법(변수의 선언)
데이터(값)의 형태 변수의 이름(변수명) 2가지 정보가 필요하다 

데이터의 형태(자료형) 변수명;

Ex) int a;

2) 변수의 값을 저장하는 방법 
변수명 = 값;
대입연산자(=) : 오른쪽의 값을 왼쪽에 저장한다
Ex) a = 10 ;

3) 선언과 동시에 값을 저장
자료형 변수명 = 값;
Ex) int a = 10;

4) 변수에 저장된 값 사용
변수명을 사용한다

5) 여러개의 변수를 선언하는 방법
자료형 변수명1, 변수명2,….

6) 변수 선언시 주의사항 
- 같은 영역( 중괄호)에서 같은 이름의 변수를 선언할 수 없다.
- 되도록 소스코드 위쪽에 한꺼번에 선언한다 
- 되도록 선언과 동시에 초기화를 한다 

7) 변수명 규칙 
(1) 띄어쓰기 불가능하다
	Int my_age 팟홀표기법, 스네이크표기법(소문자로 구성, 단어가 바뀔때 _
	Int myAge 카멜 표기법 (소문자로 시작, 단어가 바뀔때 첫문자를 대문자로 작성)
	Int MyAge 파스칼표기법 (클래스명을 만들때 주로 사용

(2)특수문자 사용이 불가능하다 (단, _언더바만 사용가능)

(3) 숫자로 시작 불가능하다 

(4) 키워드 (예약어)는 사용 불가능 하다 

(5) 웬만하면 의미있는 단어를 사용하자 

(6) 웬만하면 소문자로 시작하자
